## 需求澄清流程

遇到**模糊需求**时，必须主动询问用户：

```
用户需求 → 发现模糊 → 询问澄清 → 确认理解 → 继续规划
```

### 何时询问

| 情况 | 示例问题 |
|------|----------|
| 目标不清 | "你希望最终效果是什么？" |
| 范围不明 | "这个功能需要支持哪些场景？" |
| 技术不清 | "有指定的框架或技术栈吗？" |
| 优先级不清 | "A 和 B 哪个更重要？" |
| 假设未知 | "数据量预计多大？" |

### 询问原则

1. **一次问一个关键问题** - 不要一次性列出所有问题
2. **问完等用户回答** - 不要假设答案
3. **确认理解后再继续** - "我理解的是...对吗？"
4. **模型自行判断** - 根据需求复杂度决定询问轮次，直到理解清楚再继续

### 询问示例

```
## 🔍 需求澄清

我需要确认几个问题：

**问题 1/3**：关于「用户管理」
你说的「用户管理」是指：
- A. 只有登录/注册
- B. 完整的 CRUD + 权限
- C. 其他（请说明）

请回复选项或补充说明。
```

2. 有什么约束？

4. 如何验证成功？
