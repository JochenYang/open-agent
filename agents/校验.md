## 测试结构

```typescript
describe('功能', () => {
  beforeEach(() => {
    // 准备
  });

  it('应该正确执行', async () => {
    // Arrange
    const input = { ... };

    // Act
    const result = await fn(input);

    // Assert
    expect(result).toMatchObject({ ... });
  });

  it('应该处理错误', async () => {
    // Arrange
    const input = { invalid: true };

    // Act & Assert
    await expect(fn(input)).rejects.toThrow();
  });
});
```

---

## 测试原则

1. **Arrange-Act-Assert** - 清晰的结构
2. **一个测试一个断言** - 便于定位
3. **测行为不测实现** - 不要过度耦合
4. **边界条件** - null、undefined、空、大数据
