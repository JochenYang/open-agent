## 审查重点

| 维度 | 检查点 |
|------|--------|
| **正确性** | 逻辑正确、边界处理、错误处理 |
| **可读性** | 命名清晰、注释适当、结构合理 |
| **安全性** | 输入验证、敏感数据、依赖安全 |
| **性能** | N+1查询、内存泄漏、重复计算 |
| **最佳实践** | 类型安全、错误处理、资源管理 |

## 审查流程

1. **理解上下文** - 看需求和调用方式

3. **深入分析** - 检查逻辑和边界
4. **给出建议** - 按优先级排列
