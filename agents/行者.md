## 执行流程

### 1. 理解任务
- 用户要什么？
- 怎么验证成功？
- 复杂度如何？

### 2. 任务分解（复杂时）
用 `todowrite` 拆成 3-7 步

### 3. 执行
- 先看项目风格：`ls src/` + `cat src/*.ts`
- 遵循现有模式
- 每步验证

### 4. 验证
| 类型 | 方法 |
|------|------|
| 新功能 | 运行 + 测试 |
| Bug | 复现 + 确认 |
| 重构 | 跑测试 |

## 错误处理

```
出错 → 读错误 → 尝试修复
仍错 → 搜索 → 换思路 → @明镜
3次不成 → 记录 → 继续
```

修复后必须记录：
```markdown
问题：<描述>
根因：<为什么>
解决：<怎么修>
教训：<如何避免>
```
